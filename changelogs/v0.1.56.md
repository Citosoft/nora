### Features

- Local diff review now supports inline annotations on changed lines, a review tray in the Changes panel, and batched feedback sent back to a running agent.
- GitLab merge request details now expose a top-level review-comments popover for sending selected inline comments and their target code lines to a new agent in bulk, with new-worktree/current-branch targeting, saved handoff files to avoid dropped pasted text, clearer markdown-rendered inline comment callouts, and a manual-submit input fallback for agent handoff prompts.
- Agent settings now let missing CLIs start installation and show live install progress.
- The agent catalog now includes additional built-in CLI agents with their documented install commands.
- Developer settings now include an option to reopen the onboarding flow.
- Help now includes a content-sized Resource Monitor with live CPU and memory trend graphs plus app, agent, and terminal process-tree attribution in a separate tab.
- Opening a diff for the first time now shows a one-time guide to inline diff annotations and the review tray.
- GitLab repositories now show recent pipelines in the Forge panel with pipeline detail views for jobs and status.
- Voice settings now let dictation use either OpenAI transcription or local Whisper, with one-click model and whisper.cpp runtime installation, WAV conversion for microphone capture, clearer listening and transcribing states in the agent input, a live level bar visualizer while dictating, and a stop icon on the mic button while recording.

### Improvements

- Voice dictation no longer shows an error dialog when recording stops with no audible input; the skip is logged instead.
- Onboarding now uses clickable compact animated progress bars with distinct active/completed states, step title tooltips, step-specific headings below the bars, flatter section layouts, mandatory/optional dependency groups with compact expandable cards, copyable install commands with real clipboard feedback, app-layout theme preview choices, accent swatches, the Nora logo in the title, a separate appearance step, cleaner dependency rows with state icons before the label, and lowercase dependency labels.
- Tooltips now use a lighter high-contrast surface in dark mode so they stand out from dark app chrome.
- Diff review tray comments now show line number, text, and remove control on one compact row.
- Changed files in the Changes sidebar now show status, path, and diff stats on one compact row.
- Diff review inline comments now use primary-accent surfaces, labels, and line highlights so they stand out clearly in the diff.
- Diff review annotations now use the same inline comment styling as Forge and the Changes panel instead of a separate violet visual treatment.
- Dependency installs now override vulnerable Electron packaging transitive dependencies, pin Geist away from vulnerable Next/PostCSS packages, and declare the icon build dependency directly.
- Dependency installs now fetch Electron's executable binary reliably for local development.
- Agent CLI detection now supports Aider, Goose, Qwen Code, OpenCode, GitHub Copilot CLI, Continue CLI, Amp, and Crush.
- Agent CLI detection now supports xAI Grok Build.
- Workspace project subsection counts now appear inline in the section label instead of overlay badges.

### Fixes

- Center tab Close Others now closes agent and terminal tabs, confirming bulk agent closes once instead of leaving extra agents open.
- The macOS Help menu now includes Resource Monitor before renderer menu sync completes.
- The New Agent dialog Start agent button now uses a white bordered surface in both light and dark mode.
- The New Agent dialog task selector now defaults to No task instead of selecting the first task automatically.
- Terminal center tabs now show and restore the current working directory while keeping app git refreshes rooted at the owning project or worktree.
- Tooltips now size to their content instead of reserving a minimum width, while still capping long text at a readable max width.
- Tooltips now appear above modal dialogs instead of rendering behind the dialog overlay.
- Diff review annotations now persist per workspace in local storage so they survive workspace switches and app restarts.
- Diff review annotations now keep the original tight diff layout without inline line-number gutters or extra row spacing.
- Footer agent icons now show only detected agent CLIs that are enabled in settings.
- Agent install progress logs now hide terminal color escape codes.
- Agent detection now uses per-agent executable candidates so OpenCode is found in its default home bin directory.
- Agent settings now refresh CLI detection after an install completes so newly installed agents appear as installed.
- Footer agent icons now hide usage and account controls for CLIs that do not support those actions.
- Footer account-switch failures now open the shared error dialog instead of rendering inside the agent popover.
- Footer agent usage checks now refresh periodically without launching Codex sign-in prompts when Codex is signed out.
- Footer account controls now show Login when an agent reports no authenticated account.
- Footer account controls now launch Claude Code's browser auth flow in the background so the switch button does not stay stuck while login is pending.
- Agent icons now use packaged local logos when available with dark-mode-safe contrast, including the official Codex logo and the Claude logo for Claude Code, and fall back to favicon loading otherwise.
- Newer catalog agents now use official icon domains instead of the generic fallback placeholder.
- Newer catalog agents now show install buttons in settings using their documented install commands.
- Worktree action menus now correctly disable unavailable actions and launch workspace scripts from worktrees.
- Local commit message generation now launches the managed llama.cpp runtime with its bundled shared libraries available.
- Custom-host GitLab origin remotes now resolve from the configured GitLab host so the Forge panel detects them correctly.
- The file tree now loads large workspaces without failing when the git file list exceeds the default stdout buffer.
- Terminal typing no longer triggers background workspace refresh churn from shell output.
